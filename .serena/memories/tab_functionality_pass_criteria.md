# タブ機能の合格基準

## 合格基準（優先度順）

### 1. **タブと項目の仕様準拠** 🎯
- **総合タブ**: クリニック名, 総合評価, 費用, 特徴, 公式サイト
- **施術内容タブ**: クリニック名, 矯正範囲, 目安期間, 通院頻度, 公式サイト  
- **サービスタブ**: クリニック名, 実績/症例数, ワイヤー矯正の紹介, サポート, 公式サイト

### 2. **項目名の正しい参照** 🔗
- 指定された項目名がclinic-texts.jsonから正確に参照されること
- データが空になったり、参照エラーが発生しないこと
- 例: 「費用」項目のデータが正常に表示されること

### 3. **動的項目名変更機能** ⚡
- 項目名をランタイムで動的に変更可能
- 例: 「費用」→「コスト」への変更
- ヘッダー表示名とデータ参照の両方が正常動作

## 技術要件

### データ参照ロジック
```javascript
// 正しいデータ参照例
const cellData = this.dataManager.getClinicText(clinicCode, fieldName, '');
```

### 動的ヘッダーマッピング
```javascript
// clinic-texts.jsonの設定例
"比較表ヘッダー設定": {
  "比較表ヘッダー2": "コスト"  // 「費用」→「コスト」
}
```

### フィールド名マッピング
- UI表示名 ≠ データ参照キー の場合の適切な処理
- `getHeaderKeyForField()` メソッドでの正しいマッピング

## 確認ポイント

### テスト項目
1. 各タブクリック時に正しい項目が表示される
2. 項目名変更後もデータが正常参照される  
3. 動的ヘッダー変更が即座に反映される
4. すべてのセルにデータが表示される（空セルなし）

### デバッグ観点
- コンソールエラーの有無
- データ参照の成功/失敗
- ヘッダー名とデータキーの不整合
- CSV→JSONパース時の列オフセット問題

## 過去の問題例

### 問題1: データ参照失敗
- 原因: ヘッダー表示名をそのままデータキーとして使用
- 解決: 表示名→データキーのマッピング機能実装

### 問題2: 動的変更時の参照エラー  
- 原因: 変更されたヘッダー名で元データを参照
- 解決: 元のフィールド名での参照を維持

この基準により、UI表示とデータ参照の両方が確実に動作することを保証する。
