# ランキングサイト データ構造とビルド設計

## 目的
- クリニック共通データを一元管理し、ジャンル（メニュー）別サイトは差分のみ管理
- 追加/変更時の工数とヒューマンエラーを最小化

## ディレクトリ概要
```
public/
  global-data/               # 1つの真実のソース（マスタ集約）
    clinics.csv              # クリニックマスタ（clinic_code 起点）
    clinic-stores.csv        # 店舗マスタ
    menus.csv                # メニューマスタ
    clinic-menu-map.csv      # クリニック×メニュー 提供可否/根拠/確認日
    clinic-menu-pricing.csv  # クリニック×メニュー 価格レンジ等
    convert-global-data.js   # ↑を統合し global-data.json へ
    global-data.json         # 統合済みJSON（生成物）

  tools/
    build-available-clinics.js  # menuSlug の提供ありクリニック抽出

  <menu-slug>/
    data/
      site-common-texts.csv|json  # MV/Tipsなど共通文言（ジャンル固有）
      clinic-texts.csv|json       # 訴求コピー・特典・POINT（clinic_codeをキーに）
      ranking.csv                 # 地域×順位×clinic_code（ジャンル固有）
      available-clinics.json      # global-dataから自動生成（生成物）
      compiled-data.json          # 表示用の最終データ（生成物）
```

## キーデザイン
- 参照キーは `clinic_code` に統一（日本語名をキーにしない）
- 提供可否は `global-data/clinic-menu-map.csv` の `available` で管理
- 価格・付帯費用は `global-data/clinic-menu-pricing.csv` に集約
- 各ジャンルの `clinic-texts.*` はコピー等の差分テキストのみを保持

## ビルドフロー
1. グローバル統合
   - `node public/global-data/convert-global-data.js`
2. 提供あり抽出（任意）
   - `node public/tools/build-available-clinics.js <menu-slug> public/<menu-slug>/data`
3. ジャンルごとの最終データ生成（例）
   - `builder.js`（各ジャンル）で以下を結合して `compiled-data.json` を生成：
     - global-data.json
     - available-clinics.json（または menuToClinics[menu]）
     - clinic-texts.json（コピー類）
     - ranking.csv（並び順）
     - site-common-texts.json（共通文言）

## バリデーション（推奨）
- ranking.csv の `clinic_code` は available==='true' のみ許容
- 必須画像の存在チェック
- 価格は税込記載、限定条件は併記
- 注記（自由診療/個人差/口コミは個人の感想）必須
- evidence_url と last_verified_at の必須化（available==='true'）

## 運用例：新メニュー追加
1. `public/global-data/menus.csv` にメニュー追記
2. `clinic-menu-map.csv` に提供可否を追加（evidence_url と確認日）
3. 価格があれば `clinic-menu-pricing.csv` へ追記
4. `convert-global-data.js` 実行
5. `public/<menu>/` を作成し、`site-common-texts.*` と `clinic-texts.*` と `ranking.csv` を用意
6. `build-available-clinics.js` 実行 → available-clinics.json 生成
7. builder（または共通スクリプト）で `compiled-data.json` 生成
