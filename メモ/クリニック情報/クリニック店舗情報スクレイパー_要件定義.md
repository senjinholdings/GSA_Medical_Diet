## 1. 目的
- クリニック公式サイトのトップページURL（HP）を入力とし、同一ドメイン配下をクロールして「店舗（院）情報」を網羅的かつ構造化データとして収集する。
- 収集対象は、名称、住所、診療時間、アクセス、営業時間。

## 2. 背景
- クリニックサイトは情報構造が多様で、JSON-LDや構造化データの有無、JS描画の有無、サイトマップの品質などがまちまち。効率的で再利用可能な汎用クローラが必要。

## 3. 用語定義
- HP: 公式サイトのトップページURL。
- ルートディレクトリ: トップページ配下の同一ドメイン（サブドメイン含むかは設定で制御）。
- 院ページ（店舗ページ）: 個別院の詳細情報が主題のページ。
- 院一覧ページ: 複数の院へのリンクを集約した一覧ページ。
- 構造化データ: JSON-LD、Microdata、RDFa（schema.
org）等。

## 4. 対象範囲（スコープ）
### インスコープ
- 同一ドメイン配下（設定によりサブドメイン許可可）のHTTP/HTTPSページのクロール
- robots.txt尊重、sitemap.xmlの取り込み
- JSレンダリングが必要なページの限定的レンダリング取得（Playwright等）
- クリニックの院情報抽出（定義は後述のスキーマ）
- 出力（JSONL/CSV/SQLite）の保存
- 原ページのHTMLスナップショット保存（任意）

### アウトスコープ
- ログインが必要なページ
- PDF等のバイナリ本文パース（リンク収集は可、本文抽出は将来拡張）
- 電話予約等の自動操作

## 5. 入力仕様
- 必須: 公式サイトURL（例: https://example-clinic.jp/）
- 任意オプション:
  - 許可ドメイン/サブドメイン一覧
  - 最大クロールページ数、最大深さ
  - レート制御（並列数、間隔）
  - JSレンダリング有効化条件（URLパターン/セレクタ待機）
  - ドメイン固有セレクタ設定（YAML/JSON）

## 6. 出力仕様
- 主出力: 既存CSVへの追記（append）。
  - 出力先: `/Users/hattaryoga/Desktop/kiro_2_サイト構成分析/public/mouthpiece/data copy/出しわけSS - stores.csv`
  - 文字コード: UTF-8（BOMなし）
  - 改行: LF
  - 追記方式: 既存ファイルの最終行の次にレコードを追加。既存ヘッダは再出力しない。
  - 列構成（順序固定・必須）:
    1. `store_id`
    2. `clinic_name`
    3. `store_name`
    4. `Zipcode`
    5. `adress`（既存CSV表記に合わせて綴りはそのまま）
    6. `access`

    - `store_id`: クリニックブランドキー+枝番（例: `brand_001`）。生成規則は「ブランド短縮キー + 下位通番」。
    - `clinic_name`: ブランド/法人名（例: 「湘南美容クリニック」）。

    - `access`: 最寄駅や徒歩分などの短文（なければ `-`）。

- 付帯出力（任意・内部開発向け）: クロールログ（JSON）、抽出ログ、エラーログ、URLインデックス、HTMLスナップショット。

## 7. 収集範囲とクロールポリシー
- 同一ドメインに限定（設定でサブドメイン許可: true/false）
- パス制限: ルート配下全体。除外パターン（例: /wp-admin, /assets, /static, /privacy, /terms, /contactなど）
- サイトマップ（/sitemap.xml 等）を探索・マージ
- robots.txt尊重（Disallow遵守、Crawl-delay適用）
- 最大深さ: 既定3（設定可）。最大URL数: 既定2000（設定可）
- 優先探索: 院一覧候補URL（正規表現と日本語キーワード）
- 重複検出: 正規化URL（クエリ正規化、#除去）、正規化タイトル+アドレスハッシュ
- レート制御: 並列数既定3、バックオフ（429/5xx 時指数）

### 院一覧/院詳細ページの検出ヒューリスティクス
- URLパターン例（一覧）: /clinic/, /clinics/, /branch/, /location/, /locations/, /hospital/, /院一覧, /医院一覧, /店舗一覧, /拠点一覧, /クリニック一覧
- URLパターン例（詳細）: /clinic/.+, /hospital/.+, /branch/.+, /アクセス, /access
- キーワード（一覧検出）: 「院一覧」「医院一覧」「クリニック一覧」「店舗一覧」「拠点」「全国の」「各院のご案内」
- キーワード（詳細検出）: 「住所」「所在地」「〒」「TEL」「電話」「診療時間」「休診日」「アクセス」「最寄駅」「MAP」「地図」「予約」「Web予約」
- 正規表現:
  - 郵便番号: `〒?\d{3}-\d{4}`
  - 電話: `(?:0\d{1,4}-)?\d{1,4}-\d{4}`（内線等は後処理）
  - 曜日: `(月|火|水|木|金|土|日|祝)`

## 8. 解析ロジック優先順位
1) JSON-LD（schema.org/LocalBusiness, MedicalClinic, Organization）
2) Microdata/RDFa（同上）
3) OGP/Metaタグ（電話/住所は稀だが参照）
4) DOMヒューリスティクス（キーワード・ラベル対・テーブル構造・定義リスト）
5) パンくず経路からの院判定

重複・競合時は信頼度スコアでマージ（構造化データ>DOM抽出）。

## 9. 技術要件
- 言語: Python 3.11+
- HTTP: httpx（同期/非同期選択可）
- JSレンダリング: Playwright（ヘッドレス、ドメイン/URL条件付で実行）
- HTMLパース: selectolax or lxml + BeautifulSoup4
- 構造化データ: extruct（JSON-LD/Microdata/RDFa抽出）
- サイトマップ: sitemaps（または手実装）
- ストレージ: JSONL/CSV、SQLite（sqlite3/duckdb検討）
- キャッシュ: on-disk HTTPキャッシュ（ETag/Last-Modified活用）
- ログ: structlog or logging（JSONログ）
- CLI: Typer or argparse
- 設定: YAML（ドメイン固有セレクタ、クロール制御）
- テスト: pytest、VCR.py（HTTP録画）

## 10. 非機能要件
- 冪等性: 同一入力で同一出力（時刻フィールド除く）
- 耐障害性: 失敗URL再試行、途中再開、部分保存
- 性能: 既定で1,000ページ/10分以内（環境依存・レンダリングを除く）
- 法令/規約順守: robots.txt、利用規約に従う。個人情報配慮。
- 国際化: 文字コード自動判定（UTF-8優先、cp932等対応）

## 11. 設定・カスタマイズ
- config.yaml例（抜粋）
```yaml
start_url: https://example-clinic.jp/
allow_subdomains: false
max_pages: 1500
max_depth: 3
concurrency: 3
delay_seconds: 0.5
rendering:
  enabled: true
  url_allow_patterns: ["/clinic/", "院一覧"]
  wait_for_selectors: [".clinic-detail", "#address"]
extractors:
  domain_overrides:
    "example-clinic.jp":
      list_selectors: [".branch-list a", "a[href*='/clinic/']"]
      detail_selectors:
        name: ["h1", ".clinic-name"]
        address: [".address", "#address"]
        phone: [".tel", "a[href^='tel:']"]
storage:
  save_html_snapshot: true
  output_format: ["jsonl", "csv", "sqlite"]
```

## 14. エラーハンドリング
- HTTP 4xx/5xx: バックオフ+最大再試行3回
- タイムアウト: 接続/読み取り個別設定、既定10/20秒
- パース失敗: フォールバック（別手法）→失敗ログに詳細

### CSV追記時のバリデーション/失敗時動作
- 既存CSVのヘッダ検証（列名・順序が上記と一致）。一致しない場合は処理を中止しエラーを返す。
- 文字列内カンマや改行はCSVとして適切にクオート。
- `store_id`が空の行はスキップ（ログ出力）。
- 既存の重複チェックはデフォルト無効（明示フラグで有効化可）。
