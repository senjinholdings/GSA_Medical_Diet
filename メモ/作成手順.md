# サイト作成手順

## 1. 掲載クリニックの選定

### 選定ポイント
- **Googleリスティング広告のランキングサイトでよく掲載されているもの**
  - 競合サイトでの露出頻度を調査
  - 広告予算を投下している = 集客力があるクリニック
- **ネームバリューが高いクリニック**
  - 知名度・認知度が高い
  - ユーザーが検索しやすい
- **ASP等からアフィリエイト案件を獲得できるもの**
  - 収益化の観点から重要
  - A8.net、バリューコマース、アクセストレード等で案件確認
- **ユーザーが気になっているものと比較できること**
  - ユーザーニーズに合致
  - 検索ボリュームの多いクリニック名

### 調査方法
1. 競合ランキングサイトの掲載クリニック調査
2. Google広告「クリニック名 + 地域名」での出稿状況確認
3. ASPでの案件有無確認
4. Googleキーワードプランナーでの検索ボリューム確認

## 2. 店舗情報の取得

### 必要な店舗情報
- **クリニック名**
- **住所**
- **アクセス**（最寄り駅、徒歩時間等）
- **営業時間**
- **電話番号**
- **休診日**
- **駐車場の有無**

## 3. coolsculpting サイト完成までのステップ

### 3.1 現状確認（/public/coolsculpting/）

## 4. 医療痩身の他メニューに切り替える場合の手順

> 例：`coolsculpting` → `injection-lipolysis001`（脂肪溶解注射）、`cryolipolysis`（脂肪冷却の別構成）など。

### 4.1 ディレクトリの用意
- 既存の完成度が高いディレクトリをベースにコピー
  - 例：`public/coolsculpting/` を `public/<new-slug>/` に複製
- ルート配下の共通画像フォルダ（例：`public/images/<genre>/`）を必要に応じて作成

### 4.2 設定ファイルの更新（必須）
- `public/<new-slug>/config.js`
  - `basePath: '/<new-slug>'`
  - `assetsPath`, `dataPath`, `imagesPath` は既存踏襲（相対パス維持）
- `index.html`, `search-results/index.html` で、OG/Twitterカードのパスやロゴ参照が `/<new-slug>` 基準になっているか確認

### 4.3 データ定義の更新（CSV → JSON 変換）
- `public/<new-slug>/data/` 配下をメニュー専用に更新
  - `clinic-texts.csv`：掲載ブランド／クリニック文言・料金をジャンル用に差し替え
  - `site-common-texts.csv`：MV、比較表、Tips、注意書き等をジャンル用に差し替え
  - 類似ジャンルへ切替時も、`clinic-texts.csv` 内の「特典」「POINT1〜3（タイトル/内容）」は必ず新ジャンル仕様に更新（安全表現・注記・限定条件の併記）
- 変換スクリプトの実行（名称が同一の場合は流用）
  - `node public/<new-slug>/data/convert-clinic-texts.js`
  - `node public/<new-slug>/data/convert-site-common-texts.js`
- `compiled-data.json` の整合性確認（キー欠落・想定外の空値がないか）

### 4.4 文言・レギュレーション適合
- Tipsの3本をジャンル特性へ置換
  - 「期間と効果」：期間目安、通院頻度、施術/治療プロセスの違い
  - 「向き・不向き」：適応症例、禁忌、自己管理の要否
  - 「選び方のポイント」：そのメニューを選ぶ判断軸（通院・痛み・費用構造・サポート）
- 禁則表現の除外と注記の付与
  - 断定表現/最高表現/過度な割引訴求の抑制
  - 「自由診療」「効果・期間には個人差」「口コミは個人の感想」等の注記
  - 価格は税込、限定条件は併記（例：対象人数・期間・院限定 等）

### 4.5 画像アセットの更新
- MV画像：`public/<new-slug>/images/` に配置（`*.webp` 推奨）
- Tips画像：`public/images/<genre>/Tips1-3.webp` を用意し、`site-common-texts.*` の参照を更新
- クリニック個別のロゴ/症例画像パスの存在確認（404回避）

### 4.6 機能・表示確認
- ランキング／比較表／モーダルなど UI の体裁崩れがないか
- 検索結果ページへの遷移・パラメータ受け渡し
- レスポンシブのブレークポイント（480/768/1024）
- Service Worker/キャッシュの動作（開発時は更新反映に注意）

### 4.7 SEO・計測・公開
- タイトル/ディスクリプション/OG 画像・文言をジャンルに合わせる
- パンくず・ItemList 等の構造化データ更新
- `public/analytics-tracking.js` の読込、`vercel.json` のルーティング確認

### 4.8 チェックリスト（切替時）
- [ ] ディレクトリ複製と `basePath` の更新
- [ ] CSV→JSON 変換の実施と `compiled-data.json` 確認

- [ ] MV/Tips/ロゴのパスと実体の一致

## 5. 別カテゴリ（美容医療：ポテンツァ等）に展開する手順

> 例：`public/potenza001/` をベースに、マウスピースや医療痩身とは別カテゴリの美容医療（RFマイクロニードル等）へ展開するケース。

### 5.1 ディレクトリ・テンプレートの選定
- 既存の完成度が高いカテゴリをテンプレートとして複製（UI/データ構成が流用しやすい）
  - 例：`public/potenza001/` を新スラッグ（`/potenza002` 等）にコピー
  - あるいは `public/coolsculpting/` の CSV→JSON パイプラインを流用した新カテゴリを作成

### 5.2 設定ファイル（config.js）の整合
- `public/potenza001/config.js` の現状：
  - `basePath: '/potenza001'`
  - `assetsPath: ''`（空文字）
  - `dataPath: '/data'`（先頭スラッシュ＝絶対参照）
  - `imagesPath: ''`
- 新規カテゴリでは、アプリの読み込み仕様に合わせて以下のいずれかに統一：
  - 相対参照で統一する場合：`assetsPath: '.'`, `dataPath: './data'`, `imagesPath: './images'`
  - 既存仕様を踏襲する場合：`dataPath` の絶対参照に合わせて `app.js` のフェッチ先を確認

### 5.3 データモデルの選択と整備
- A案：CSV→JSON 変換を導入
  - `data/clinic-texts.csv`, `data/site-common-texts.csv` を用意
  - 変換スクリプト（`convert-*.js`）を配置し、更新フローを標準化
- B案：`compiled-data.json` を直接運用
  - スキーマ（キー構成）を `app.js` 側の期待値に合わせて更新
- どちらの案でも、ブランド/クリニックの掲載方針（選定基準・レギュ対応）を反映

### 5.4 コンテンツ設計（ポテンツァ例）
- Tips（3本）の観点例：
  - 「期間と効果」：施術回数の目安（例：3〜5回）、効果の発現タイミング、肌質/悩みによる個人差
  - 「向き・不向き」：ニキビ跡/毛穴/たるみ等の適応、金属アレルギーや皮膚疾患の注意
  - 「選び方のポイント」：針長・出力・モード、麻酔/鎮痛、ダウンタイムやアフターケア、症例数
- 比較・ランキング軸：料金（税込/初回/コース）、通いやすさ、症例/カウンセリング体制、保証やアフターケア

### 5.5 表現・レギュレーション
- 医療広告ガイドラインに準拠：
  - 断定・誇大表現の回避（「必ず」「確実に」等）
  - 価格は税込表記、限定条件の併記（人数/期間/対象等）
  - 「自由診療」「効果・期間には個人差」「症例写真は一例」「口コミは個人の感想」等の注記
  - リスク・副作用（赤み・腫れ・内出血・熱感 等）の掲示

### 5.6 画像・アセット
- MV/バナー/アイコン/症例写真をカテゴリ専用に差し替え（WebP推奨、解像度最適化）
- `site-common-texts.*` の Tips 画像パスと実体を一致させる
- ロゴ・クリニック画像の404回避（存在チェック）

### 5.7 動作・品質確認
- ランキング/比較表/詳細表示のUI崩れ確認
- ルーティング（`basePath`）とデータフェッチ先の一致
- レスポンシブ（480/768/1024）・カルーセル・検索結果の動作
- Service Worker/キャッシュの更新反映

### 5.8 SEO・公開
- タイトル/ディスクリプション/OG文言・画像をカテゴリに適合
- 構造化データ（Breadcrumb/ItemList/FAQ等）を更新
- 計測タグ読込、`vercel.json` のルーティング・キャッシュ確認
